Creates an input field. For input fields of type "text" or "password" the label associated with the field can be specified inside the <o2 input> tag. This allows for very compact and readable template code. For radio button and checkbox groups, see and . Another way to write forms is to use the tag - the form fields are then put inside table cells. rule - a rule that must be obeyed (see the tag for the rules) ruleMsg - the text that will be displayed to the user in case the rule wasn't obeyed counterId - if you specify a counter ID, you should also have an tag name label - the text associated with the input field (applies to "text" and "password" fields) labelClass - extra class for the label accesskey - if label is specified, you can also specify an accesskey focus [0] - should this field receive focus when the page loads? (At most one focused field per page) containerStyle - the style of the generated tr or div tag that surrounds the input tag numberFormat - if the original number in the input field should be formatted in a special way (ex: numberFormat="0,##") multilingual - should the user be able to edit the input field in more than one language? (Also see ) Normal html attributes In order to look good, for example together with multiple fields or other radio button or checkbox groups, the HTML output by the input tag must be styled with CSS, or the tag could be used.
