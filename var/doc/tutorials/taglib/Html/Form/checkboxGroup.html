Creates a checkbox group. This example has not been styled. Another way to write forms is to use the tag - the form fields are then put inside table cells. name - the name of the checkboxes / input tags values - which checkboxes should be checked when the page loads (comma-separated list or array reference) label - a text associated with the entire checkbox group labelClass - extra class for the label focus [0] - should this checkbox group receive focus when the page loads? (At most one focused field per page) display - must be one of {inline, block} - if the checkboxes should be displayed on the same row (inline) or one checkbox per line (block). In the CSS example below the default is set to block (last rule). rule - a rule that must be obeyed (see the ) ruleMsg - the text that will be displayed to the user in case the rule wasn't obeyed content - checkbox tags value - the value of the checkbox - same as the value attribute of the generated input tag. label - a text associated with this particular checkbox labelClass - extra class for the label accesskey checked [0] - same as the "value" attribute of the checkboxGroup tag lineBreakAfter [0] - adds a line break after the checkbox (if "display" is "inline") id - by default a concatenation of the name and value will be used as the ID In order to look good, for example together with multiple fields or other radio button or checkbox groups, the HTML output by checkboxGroup and checkbox tags must be styled with CSS, or the tag could be used.
