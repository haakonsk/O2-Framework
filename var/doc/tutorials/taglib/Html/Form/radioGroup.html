Creates a group of radio buttons. This example has not been styled. Another way to write forms is to use the tag - the form fields are then put inside table cells. name - the name of the radio buttons / input tags value - which radio button should be checked when the page loads label - a text associated with the entire radio button group labelClass - extra class for the label focus [0] - should this radio group receive focus when the page loads? (At most one focused field per page) display - must be one of {inline, block} - if the radio buttons should be displayed on the same row (inline) or one radio button per line (block). In the CSS example below the default is set to block (last rule). content - radio tags value - the value of the radio button - same as the value attribute of the generated input tag. label - a text associated with this particular radio button labelClass - extra class for the label accesskey checked [0] - same as the "value" attribute of the radioGroup tag lineBreakAfter [0] - adds a line break after the radio button (if "display" is "inline") rule - a rule that must be obeyed (see the ) ruleMsg - the text that will be displayed to the user in case the rule wasn't obeyed In order to look good, for example together with multiple fields or other checkboxes or radio button groups, the HTML output by radioGroup and radio tags must be styled with CSS, or the tag could be used.
