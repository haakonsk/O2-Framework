Next numPerPage - the number of results to display per page. The last page may contain less results than this number. totalNumResults - the total number of results returned by this specific search. Needed so that the module is able to calculate the total number of pages and provide links accordingly. elements - a call to a function that should return the elements/results to display on the current result page. Two of the arguments in the function call must be $skip and $limit. They don't have to be actual values, since they're given values dynamically with a regular expression, depending on which page is the current one. The elements are available in the array ref variable $paginationResults. If the "elements" attribute isn't given, the elements of $paginationResults will be numerical indices instead of elements. content - go through the results and add link tags. The results for the current page are available in the array ref variable $paginationResults. See below for the link tags that are available. Other variables that are available inside <o2 pagination> are $paginationFirstIndex and $paginationLastIndex. They are the index/position of the first and last result on the current page, respectively. linkSeparator - a string used to separate the links used to navigate between result pages. content - more specific link tags, see below. If omitted, a previous link, numeric links and a next link are included. If the result is small enough to be displayed on just one page, then this tag doesn't generate any output. No content is generated by this tag if there are no result pages before the current page. class [paginationPrevious] - The class attribute of the generated <a> tag. content - the link text. The default is to use the language variable o2.pagination.previous. Other html attributes - passed along to the <a> tag. No content is generated by this tag if there are no more result pages after the current page. class [paginationNext] - The class attribute of the generated <a> tag. content - the link text. The default is to use the language variable o2.pagination.next. Other html attributes - passed along to the <a> tag. No link to the current result page is generated. But the number of the current result page is still displayed between the numeric links to the previous and next page. The class "paginationCurrentPage" is given to this element, making it possible to style differently than the links surrounding it. numBefore [10] - the maximum number of previous-links to display. If numBefore is 2 and the current page is number 4, then links to result pages 2 and 3 are displayed. numAfter [10] - the maximum number of next-links to display. If numAfter is 3 and the current page is number 4, then links to result pages 4, 5 and 6 are displayed. But if the total number of result pages is just 5, then only links to 4 and 5 are displayed. linkSeparator - a string used to separate the links used to navigate between result pages. Defaults to the linkSeparator attribute in the <o2 paginationNavigation> tag. class [paginationNumeric] - The class attribute of the generated <a> tags.
