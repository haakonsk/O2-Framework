Language specific text is stored under var/resources/ where each language has its own directory. To get the correct text for the currently chosen language (active locale), use this tag (getString) or the global \$lang variable. commonFiles.commonText.msgFormError Another way of writing this is as follows: $lang->getString("commonFiles.commonText.msgFormError") If the language string contains variables, their values are passed as parameters in the getString tag. Suppose, for example there's a language variable named subscriberPrintMapInfo in the file Shop/MapPurchase.conf. In swedish the value of this language variable is "Du har kvar \$numMapsLeft av \$initialNumberOfMaps nedladdningar/utskrifter, Abonnemanget går ut \$expires". You can print the variable in one of two ways. Either: Shop.MapPurchase.subscriberPrintMapInfo or $lang->getString("Shop.MapPurchase.subscriberPrintMapInfo", numMapsLeft => 98, initialNumberOfMaps => 100, expires => "2007-07-18") content - the resource path Variable names and values (See example above) See also .
